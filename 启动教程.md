# Fitness健身记录 — 启动教程

## 文档说明

本项目有两种使用方式，**不要混用访问地址**：

| 场景 | 适用情况 | 前端 | 后端 | 浏览器访问 |
|------|----------|------|------|------------|
| **日常开发** | 在 Cursor 里改代码、调试 | `npm run dev`（热更新） | `spring-boot:run -Pdev` | http://localhost:3000/fitness/ |
| **正式部署** | 演示、交作业、与线上一致 | `npm run build` 打进 WAR | Tomcat 运行 WAR | http://localhost:8080/fitness/ |

> 前端为 React 19 + Next.js 15 静态导出 SPA。正式环境无需 Node.js；开发改前端时需要 Node.js。

## 一、首次环境准备（开发与部署共用）

### 1. 初始化数据库

**1）确保 MySQL 服务已启动**

**2）登录 MySQL：**

```bash
mysql -u root -p123456 --default-character-set=utf8mb4
```

**3）依次执行 SQL 脚本：**

```sql
-- 建表
source D:/test-codes/fitness/sql/schema.sql;

-- 插入测试数据（26个动作 + 管理员账号）
source D:/test-codes/fitness/sql/data.sql;

-- 验证
USE fitness;
SELECT COUNT(*) FROM exercise;   -- 应显示 26
SELECT * FROM user;              -- 应显示 admin

EXIT;
```

### 2. 确认数据库连接配置

文件：`src/main/resources/application.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/fitness?useUnicode=true&characterEncoding=UTF-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 123456
```

### 3. 首次安装前端依赖（仅开发/打包时需要）

```powershell
cd D:\test-codes\fitness\frontend
npm install --legacy-peer-deps
```

## 二、在 Cursor 里日常开发（推荐）

改代码时**不要**每次 `npm run build` + 打 WAR + 重启 Tomcat。

### 一次性准备

1. 打开项目后安装推荐扩展：**Extension Pack for Java**、Tailwind CSS。
2. 完成上文「首次环境准备」中的数据库与 `npm install`。

### 一键启动

| 操作 | 快捷键 / 菜单 |
|------|----------------|
| 启动后端 + 前端 | `Ctrl+Shift+B`（默认构建任务） |
| 或 | `终端` → `运行任务` → **dev: 全栈（后端 + 前端）** |

启动后访问：

| 用途 | 地址 |
|------|------|
| **改前端页面**（保存即刷新） | http://localhost:3000/fitness/ |
| 后端 API | http://localhost:8080/fitness/ |

### 分别启动 / 调试

| 需求 | 操作 |
|------|------|
| 只改 Java，可断点 | `Ctrl+Shift+D` → **调试后端** → F5 |
| 只改前端 | 任务 **dev: 前端 (Next.js)**，或 **启动前端 Dev** |
| 前后端一起 | **全栈开发（调试后端 + 前端）** |

### 改代码后要做什么？

| 修改位置 | 需要做的 |
|----------|----------|
| `frontend/src/**` | 保存即可，浏览器自动刷新 |
| `src/main/java/**` | DevTools 自动重启后端（约几秒），或重启「dev: 后端」任务 |
| 演示 / 交作业 / Tomcat | 见下文 **「三、正式部署」** |

### 命令行等价方式（不用 Cursor 任务时）

```powershell
# 终端 1：后端（带 dev 配置，内置 Tomcat）
cd D:\test-codes\fitness
$env:JAVA_HOME = "D:\Develop\jdk14"
D:\Develop\apache-maven-3.9.11\bin\mvn spring-boot:run -Pdev -Dmaven.test.skip=true

# 终端 2：前端
cd D:\test-codes\fitness\frontend

```

访问：http://localhost:3000/fitness/

## 三、正式部署（Tomcat）

将**最新前端静态资源**与**后端**一并打入 WAR，由 Tomcat 在 **8080** 提供完整站点。与日常开发的区别见文首表格。

### 部署前检查

- [ ] MySQL 已启动，`fitness` 库与数据已初始化
- [ ] `application.yml` 中数据库账号密码正确
- [ ] 已安装 Node.js（用于 `npm run build`）
- [ ] 若 Tomcat 已在运行，先停止（见下文「停止 Tomcat」）

### 方式 A：在 Cursor 中操作（推荐）

按顺序执行 **终端 → 运行任务**：

| 顺序 | 任务名称 | 说明 |
|------|----------|------|
| 1 | **release: 前端构建** | 导出到 `src/main/resources/static/` |
| 2 | **release: 打包 WAR** | 生成 `target/fitness.war` |
| 3 | 手动部署 Tomcat | 见下方「停止 → 部署 → 启动」 |

### 方式 B：命令行一条龙

```powershell
# ---------- 1. 构建前端（必须，否则 WAR 内是旧页面）----------
cd D:\test-codes\fitness\frontend
npm run build

# ---------- 2. 打包 WAR（不要用 -Pdev）----------
cd D:\test-codes\fitness
$env:JAVA_HOME = "D:\Develop\jdk14"
D:\Develop\apache-maven-3.9.11\bin\mvn clean package -Dmaven.test.skip=true

# 产物：D:\test-codes\fitness\target\fitness.war
```

### 停止 Tomcat → 部署 WAR → 启动 Tomcat

**PowerShell：**

```powershell
$env:CATALINA_HOME = "D:\Develop\apache-tomcat-9.0.118"
$env:JRE_HOME = "D:\Develop\jdk14"

# 停止
& "$env:CATALINA_HOME\bin\shutdown.bat"
Start-Sleep -Seconds 3

# 删除旧部署（避免缓存旧版本）
Remove-Item -Recurse -Force "$env:CATALINA_HOME\webapps\fitness" -ErrorAction SilentlyContinue
Remove-Item -Force "$env:CATALINA_HOME\webapps\fitness.war" -ErrorAction SilentlyContinue

# 复制新 WAR
Copy-Item -Force "D:\test-codes\fitness\target\fitness.war" "$env:CATALINA_HOME\webapps\"

# 启动
Start-Process "$env:CATALINA_HOME\bin\startup.bat" -WorkingDirectory "$env:CATALINA_HOME\bin"
```

**或双击：** 先 `shutdown.bat`，再拷贝 WAR 到 `webapps\`，再 `startup.bat`。

> 若 `shutdown.bat` 提示 `CATALINA_HOME` 未定义，请先执行上面的 `$env:CATALINA_HOME = "..."`，或在本机系统环境变量中配置。

启动成功时，Tomcat 窗口会出现：

```
Server startup in [xxx] milliseconds
```

### 只改了部分代码时，最少要做哪些？

| 修改内容 | 最少步骤 |
|----------|----------|
| 只改 `frontend/src/**` | `npm run build` → `mvn package`（可省略 `clean`）→ 替换 WAR → 重启 Tomcat |
| 只改 `src/main/java/**` | `mvn package` → 替换 WAR → 重启 Tomcat |
| 前后端都改 | `npm run build` → `mvn clean package` → 替换 WAR → 重启 Tomcat |

> 只改 Java 却未重新 `npm run build` 时，WAR 内前端仍为旧版，一般不影响后端接口调试。

### 停止 Tomcat

```powershell
$env:CATALINA_HOME = "D:\Develop\apache-tomcat-9.0.118"
& "$env:CATALINA_HOME\bin\shutdown.bat"
```

或双击 `D:\Develop\apache-tomcat-9.0.118\bin\shutdown.bat`（需已配置环境变量）。

## 五、常见问题

### Tomcat 窗口闪退

用文本编辑器打开 `startup.bat`，在文件末尾添加一行 `pause`，保存后重新双击。

### 启动日志报 `Ambiguous mapping`

Spring MVC 路径冲突。检查控制器中是否有重复的 `@GetMapping` 路径。

### 数据库连接失败

- 确认 MySQL 已启动
- 确认已执行 `sql/schema.sql` 与 `sql/data.sql`
- 检查 `application.yml` 中的 `username`、`password`

### 端口 8080 被占用

```powershell
netstat -ano | findstr :8080
taskkill /F /PID <上一步看到的PID>
```

### 正式环境页面样式错乱 / 仍是旧页面

1. 确认已执行 **`npm run build`**（不是 `npm run dev`）
2. 确认 `src/main/resources/static/_next/` 存在且时间戳为最新
3. 删除 Tomcat 下旧的 `webapps/fitness` 文件夹后再部署 WAR
4. 浏览器 **Ctrl+F5** 强制刷新

### `spring-boot:run` 与 Tomcat 的区别

| 命令 | 用途 |
|------|------|
| `mvn spring-boot:run -Pdev` | **仅本地开发**，内置 Tomcat + DevTools |
| `mvn clean package`（无 `-Pdev`） | **正式打包**，供外置 Tomcat 部署 |
| Tomcat `webapps/fitness.war` | **正式运行**，与答辩/演示环境一致 |

### 开发时访问 8080 和 3000 有什么区别？

- **3000**：Next 开发服务器，改前端立即生效，API 代理到 8080。
- **8080（开发模式）**：仅 Spring Boot API；若未 `npm run build`，静态页可能是旧构建结果。
- **8080（Tomcat）**：前后端均为发布版本，与正式部署一致。

### 日常开发页面空白 / 没有数据 / 提示网络失败

1. **必须同时启动两个服务**：`Ctrl+Shift+B`（后端 8080 + 前端 3000），只开前端会读不到数据库。
2. **必须在 3000 端口访问**：http://localhost:3000/fitness/（不要只用 8080 看开发页面）。
3. **确认 MySQL 已启动**，且后端日志无数据库连接错误。
4. 在浏览器直接打开 http://localhost:8080/fitness/exercise/api/list ，应返回 JSON；若这里正常而 3000 不正常，**重启前端** `npm run dev`（修改 `next.config.ts` 后需重启）。
5. 若 8080 被 Tomcat 占用，先 `shutdown.bat` 再启动开发后端。

### 开发时控制台出现 `params are being enumerated`

**现象**：在 Cursor 里用 `npm run dev` 开发时，操作页面上的 `<select>` 或移动鼠标，控制台反复出现：

`params are being enumerated. params should be unwrapped with React.use()...`

**原因**：Next.js 15 的路由 `params` 是 Promise，不能被 `Object.keys()` 等方式枚举。Cursor 元素检查器在鼠标移动时会扫描 React 组件 props，从而触发该提示。**不是业务代码错误，也不影响生产部署。**

**项目已做处理**：开发环境会在 `layout.tsx` 中加载 `public/suppress-params-warning.js`（路径带 `/fitness` 前缀），过滤该已知噪音。

**若仍出现或页面白屏**：

1. 确认访问地址为 http://localhost:3000/fitness/（注意 `basePath`）
2. 硬刷新浏览器（Ctrl+Shift+R）
3. 清理前端缓存并重启 dev：

```powershell
cd D:\test-codes\fitness\frontend
Remove-Item -Recurse -Force .next -ErrorAction SilentlyContinue
npm run dev
```

**编写动态路由时**（如 `/exercise/[id]`）：服务端页面使用 `const { id } = await params`；客户端组件使用 `React.use(params)`，不要直接同步访问或展开 `params`。
