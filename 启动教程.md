## 环境信息

| 软件 | 版本 | 本机路径 |
|------|------|----------|
| JDK | 14 | `D:\Develop\jdk14` |
| Maven | 3.9.11 | `D:\Develop\apache-maven-3.9.11` |
| MySQL | 8.0.33 | `root / 123456` |
| Tomcat | 9.0.118 | `D:\Develop\apache-tomcat-9.0.118` |

## 第一步：初始化数据库

**1. 确保 MySQL 服务已启动**

**2. 打开命令行终端，登录 MySQL：**

```bash
mysql -u root -p123456 --default-character-set=utf8mb4
```

**3. 依次执行 SQL 脚本：**

```sql
-- 建表
source D:/test-codes/fitness/sql/schema.sql;

-- 插入测试数据（26个动作 + 管理员账号）
source D:/test-codes/fitness/sql/data.sql;

-- 验证
USE fitness;
SELECT COUNT(*) FROM exercise;   -- 应显示 26
SELECT * FROM user;              -- 应显示 admin

-- 退出
EXIT;
```

---

## 第二步：确认数据库连接配置

文件位置：`src/main/resources/application.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/fitness?useUnicode=true&characterEncoding=UTF-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 123456
```

---

## 第三步：编译打包

```bash
cd D:/test-codes/fitness
D:\Develop\apache-maven-3.9.11\bin\mvn clean package -Dmaven.test.skip=true
```

成功后WAR包位于：`target/fitness.war`

---

## 第四步：部署到 Tomcat

```bash
# 复制 WAR 包到 Tomcat
copy D:\test-codes\fitness\target\fitness.war D:\Develop\apache-tomcat-9.0.118\webapps\
```

> 如果Tomcat之前已启动，先执行 `D:\Develop\apache-tomcat-9.0.118\bin\shutdown.bat` 停止后再部署。

---

## 第五步：启动 Tomcat

直接双击运行 `D:\Develop\apache-tomcat-9.0.118\bin\startup.bat`

会弹出一个命令行窗口，等待出现以下日志表示启动完成：

```
Server startup in [xxx] milliseconds
```

> 如果窗口闪退看不到日志，用文本编辑器打开 `startup.bat`，在末尾添加一行 `pause`，保存后重新双击。

## 常见问题

### Tomcat窗口闪退

用文本编辑器打开 `startup.bat`，在文件末尾添加一行 `pause`，保存后重新双击。窗口会停住显示错误信息。

### 启动日志报 `Ambiguous mapping`

说明有Spring MVC路径冲突。检查控制器类中是否有重复的 `@GetMapping` 路径。

### 数据库连接失败

- 确认MySQL服务已启动
- 确认 `fitness` 数据库已创建并执行了SQL脚本
- 检查 `application.yml` 中的 `username` 和 `password`

### 端口被占用

```bash
# 查看谁占用了8080端口
netstat -ano | findstr 8080

# 强制结束进程（替换为实际PID）
taskkill /F /PID <PID>
```

### 重新部署步骤

如果修改了代码需要重新部署：

```bash
# 1. 停止Tomcat（双击 shutdown.bat）
# 2. 重新编译
cd D:/test-codes/fitness
D:\Develop\apache-maven-3.9.11\bin\mvn clean package -Dmaven.test.skip=true

# 3. 删除旧部署
rmdir /s /q D:\Develop\apache-tomcat-9.0.118\webapps\fitness
del D:\Develop\apache-tomcat-9.0.118\webapps\fitness.war

# 4. 复制新WAR包
copy target\fitness.war D:\Develop\apache-tomcat-9.0.118\webapps\

# 5. 启动Tomcat（双击 startup.bat）
```
